# CSS에서의 Height 100% 동작 원리 (The Standard of `height: 100%`)

웹 퍼블리싱이나 리액트 컴포넌트 스타일링을 하다 보면 `width: 100%`는 의도한 대로 꽉꽉 잘 차는데, `height: 100%`는 유독 먹히지 않거나 내용물(content) 크기만큼만 쪼그라드는 현상을 자주 겪게 됩니다.

그 이유와 정확한 동작 원리를 정리합니다.

---

## 1. Width와 Height의 기본 동작 차이

### Width (너비)
*   Block-level 요소(`<div>`, `<section>` 등)의 `width` 기본값은 `auto`입니다.
*   하지만 브라우저는 Block 요소의 `width: auto`를 **"부모 요소의 남은 가로 공간을 최대한 모두 차지하라(100%)"** 로 해석합니다. 
*   따라서 가로 크기는 굳이 `100%`를 주지 않아도 알아서 화면 끝까지 뻗어나갑니다.

### Height (높이)
*   Block-level 요소의 `height` 기본값 역시 `auto`입니다.
*   하지만 브라우저는 `height: auto`를 전혀 다르게 해석합니다. **"네 안에 들어있는 내용물(자식 요소들, 텍스트)의 높이만큼만 최소한으로 크기를 늘려라"** 로 해석합니다.
*   브라우저의 스크롤은 기본적으로 세로 방향이므로, 내용물이 많아지면 밑으로 길어지는 것이 자연스러운 문서(Document)의 흐름(Flow)이기 때문입니다.

---

## 2. `height: 100%`의 진짜 의미

CSS에서 퍼센트(`%`) 단위는 무조건 **"자신을 감싸고 있는 직계 부모(Direct Parent) 요소"**를 기준으로 계산됩니다.

*   `height: 100%` = **"내 직계 부모가 가진 높이값과 똑같은 크기를 가질게!"**

### 문제 발생 시나리오
```html
<body> <!-- 기본 높이: 내용물만큼 (auto) -->
  <div class="parent"> <!-- 기본 높이: 내용물만큼 (auto) -->
    <div class="child" style="height: 100%"> 
      <!-- 부모(auto)의 100%는 얼마일까? 기준이 없으므로 얘도 결국 auto가 됨 -->
    </div>
  </div>
</body>
```
부모 요소의 높이가 명시적인 값(`px`, `vh` 등)이 아니라 `auto`라면, 자식 요소에게 `height: 100%`를 주더라도 **부모 높이의 실체를 알 수 없으므로 무시되고 결국 자식 자신의 내용물 크기(auto)로 렌더링**됩니다.

---

## 3. 해결책: 릴레이 상속 (The Chain of Heights)

어떤 자식 요소가 화면 전체 세로 길이에 맞춰 `height: 100%`를 온전히 누리려면, **최상단(html)부터 해당 요소에 이르기까지 모든 부모가 명시적인 높이를 릴레이처럼 전달**해주어야 합니다.

### React / SPA 환경에서의 모범적인 CSS 세팅
React 프로젝트(특히 CRA나 Vite 기반)에서는 루트 CSS(`index.css`나 `App.css`)에 아래와 같은 글로벌 세팅을 깔아두는 것이 관행입니다.

```css
/* 1. 브라우저 창(Viewport) 전체를 기준점으로 잡음 */
html, body {
  height: 100%;
  margin: 0;
}

/* 2. React가 렌더링되는 최상위 DOM 노드도 100%를 상속받음 */
#root {
  height: 100%; /* 또는 min-height: 100vh; */
}
```

이렇게 `html` -> `body` -> `#root` 까지 화면 전체 높이(`100vh`)의 기준이 내려왔다면, 그 내부의 리액트 컴포넌트들(`App`, `Page`, `Container`)은 이제 안심하고 `height: 100%`를 사용하여 부모의 높이를 쭉쭉 뻗어 나갈 수 있습니다.

---

## 4. 모던 웹 레이아웃에서의 Flex Box 결합

요즘은 `height: 100%`만으로 레이아웃을 잡기보다는, `flex-direction: column`과 결합하여 훨씬 유연하게 화면을 구성합니다.

```tsx
// Room.tsx 예시
<div style={{ height: "100%", display: "flex", flexDirection: "column" }}>
  <header>고정 높이 헤더</header>
  
  {/* 남는 공간을 모두 차지하는 메인 영역 */}
  <main style={{ flex: 1 }}>
    비디오 그리드
  </main>
</div>
```

**요약하자면:**
`height: 100%`는 **"부모로부터 물려받은 세로 공간의 캔버스"**를 활짝 펼치는 역할을 합니다. 이 캔버스가 제대로 펼쳐지려면, 반드시 최상단 부모부터 명시적인 높이값(`100vh` 등)이 끊기지 않고 전달되어 내려와야 합니다.
